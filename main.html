Types de snippet

codesnip dispose de 4 types de snippet qui permettrons de disposer le contenu selon un format particulier dans le document HTML :
  • Le snippet pour le texte simple
  • Le snippet pour le HTML
  • Le snippet pour le CSS
  • Le snippet pour le JavaScript
1. Snippet de texte
Ce snippet permet de disposer un contenu au format texte simple et ne possède pas de coloration syntaxique.
Il s'avère utile dans la mesure où souhaiterai illuster un certain texte pur.
Ceci est un exemple de snippet de texte simple qui peut pourrait s'avérer très utile dans certains contextes. --- CODESNIP---
2. Snippet pour HTML
Ce snippet permet de disposer un contenu au format HTML avec coloration syntaxique.
Il faudrait cependant souligner qu'il n'est pas possible de combiner le HTML et CSS ou JavasCript dans le même snippet parceque chaque snippet a son propre algorithme d'analyse pour detecter certains élements spécifiques à ce langu

Hallo codesnip

Ceci est un exemple de snippet pour HTML

3. Snippet pour CSS
Ce snippet permet de disposer un contenu au format CSS avec coloration syntaxique des élements basiques spécifiques au langage CS
@media (min-width : 1280px){ body { max-width : 1280px; margin: 0 auto; } }
4. Snippet pour JavaScript
Ce snippet permet de disposer un code JavaScript avec coloration syntaxique de certains élements du langage.
Il faut souligner qu'il n'y a pas une très grande palette d'élements de syntaxe pris en compte. Toutefois, ce model de snippet prend en compte un certain nombre d'élements de syntaxe necessaires pour un JavaScript valide et comprehensif.

Manipulation JavaScript

Un snippet valide doit au-moins avoir une structure html semblable à ce qui suut :
Endroit principal ou disposer le contenu à analyser
L'attribut csnip-lang permet de spécifier la façons dont on aimerait considérer et analyser le contenu qui se trouve dans la balise ayant la classe csnip-body

En JavaScript, il faudrait invoquer la fonction csnip() , laquelle retourne un objet sur lequel il est possible d'appler les methodes suivante pur effectuer un traitement :
csnip(arg)
La cette fonction est la principale function sur la quelle seront appelées les autres méthodes qui suivent.
Suivant le contexte, cette méthode prend en argument : soit un noeud du DOM, soit une chaîne de caractère faisant réference à un selecteur
Il est important de savoir que les élements ciblés par selecteur oupar noeud du DOM doivent faire reférence à un ou plusieurs snippets de votre document html actuel.
csnip('[csnip="js"]') //Exemple avec un selecteur csnip(document.getElementById("mySnippetId")) // Exemple avec un noeud du DOM
csnip().ready(callback)
Cette méthode est un mécanisme de sécurité qui permet de n'exécuter un certain nombre de manipulations que si le DOM est chargé et disponible pour la manipulation.
Elle prend en argument une function qui sera executée dès que le DOM sera prêt aux manipulations
csnip().ready(function(){ console.log("je suis prêt !"); csnip().docparse(); // A voir plus bas... })
csnip(arg).parse(content, lang)
Cette méthode prend en argument deux élements :

1. Un contenu à analyser et qui sera mis dans le snippet ciblé.
2. Un langage cible dans lequel le contenu précedent doit être analysé. Ces languages cible sont : html, js, css, text.
csnip('[csnip="js"]').parse("var name = 'John Doe'", "js") /* Analyse le snippets trouvé avec la chaîne de selecteur specifiée avec le contenu "var name = 'John Doe'" en tant que JavaScript et met le résutat de l'analyse dans les snippets ciblés. */

Il faudrait souliger que les arguments passés à cette methode sont optionnelles.
Si on omet, les snippets ciblés seront analysé en fonction de l'attribut csnip-lang qui sera trouvé sur ceux-ci.
csnip(arg).update(content, lang)
Cette méthode fonctionne un peu comme parse() mais avec la différence que celle-ci met à jour le snippet ciblé avac le nouveau contenu fourni en premier argument
csnip('[csnip="js"]').update("let newContent = 'i am a new content !'", "js"); /* Met à jour (ou remplace) l'anien contenu par le nouveau passé en premier argument et l'analyse dans le langage cible passé en deuxième argument */

Il faudrait noter que la langue cible est optionnelle. La langue sera automatiquemet détectée en fonction des attributs du snippet
csnip(arg).styles({})
Cette méthode est employé pour styliser l'apparence du snippet.
Elle prend en argument un objet qui répresente les styles CSS à appliquer au snippet.
csnip('[csnip="js"]').styles({ 'border-radius' : '5px', 'margin' : '2rem 0', 'background-color' : '#eee' }); /* Stylise le snippet cyblé avec les propriétés CSS spécifiés. */
csnip(arg).docparse()
Cette méthode analyse tout le document html actuel et détecte tous les snippets puis les analyse en fonction des attributs spécifiques à chacun.

Il est recommandé d'utilser cette méthode après que le DOM soit prêt.
csnip().ready(function(){ csnip().docparse(); }); /* Attend la disponibilité du DOM, Analyse tout le document html et les snippets présents. */

Personnalisation d'élements de langage

Dans le contexte où vous souhaiteriez modifier l'apparence par défaut des snippets en général et les couleurs de certains élements typiques d'un langage en particulier, il est recommandé de procéder avec le plus d'attention possible à la modification du fichier codesnip.js fournit avac cette libraire.

C'est dans ce fichier qu'il sera possible de modifier le CSS de certains élement de laguage et les modifications doivent être effectuées uniquement sur les valeurs des propriétés.

Il n'est cependant pas recommandé de modifier les noms des classes ou des propriétés se trouvant dans ce fichier, par crainte d'incohérences entre le code de la libraire et le CSS de ce fichier et des erreurs de rendu.
